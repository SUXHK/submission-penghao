## 4. 被采纳的 AI 建议

| 建议                            | 采纳原因                       | 落地结果                          |
| ------------------------------- | ------------------------------ | --------------------------------- |
| Spring Boot 分层架构            | 企业标准，符合题目要求         | Controller/Service/Repository三层 |
| 9状态流转机（含REOPENED共10种） | 完整覆盖业务链路               | 含校验规则的完整状态机            |
| TanStack Table                  | 项目已有依赖，排序分页开箱即用 | 客户端排序+分页                   |
| Sheet面板设计                   | 保持列表上下文，交互流畅       | 右侧滑出Sheet，URL同步状态        |
| 客户端筛选                      | 数据量小，即时响应             | 切换tab无延迟                     |

## 5. 被拒绝或修正的 AI 建议

| 建议                 | 问题               | 人的判断 / 修正                       |
| -------------------- | ------------------ | ------------------------------------- |
| H2内存数据库         | 不符合企业场景     | 改用MySQL 8 + Docker                  |
| 服务端分页           | 原型阶段数据量小   | 改用客户端分页                        |
| 简化5状态流转        | 不够严谨           | 坚持9状态完整流转（含REOPENED共10种） |
| 渐变卡片统计栏       | 多次不满意         | 反复迭代至当前设计                    |
| defaultValue输入方案 | 切换字段时值不更新 | 加key={field}强制重建                 |

## 6. AI 出错记录

| 问题类型   | 表现                       | 发现方式               | 修正方式                    |
| ---------- | -------------------------- | ---------------------- | --------------------------- |
| 理解偏差   | 统计栏设计方向反复调整     | 用户反馈"不好看"       | 多轮迭代+选项提问           |
| 过度设计   | 侧栏导航栏+顶栏重复        | 用户指出"顶部已有导航" | 移除侧栏                    |
| 代码错误   | defaultValue切换字段不更新 | 保存后数据未变更       | 添加key prop                |
| 测试遗漏   | E2E测试Sheet关闭按钮找不到 | Playwright超时         | 改用force click+page reload |
| 文档不准确 | 端口号与实际不符           | 启动时端口冲突         | 统一修改端口配置            |
| 架构设计缺陷 | AI建议异步生成+前端轮询导致大量无效请求 | 用户反馈"一直调用了10次" | 改为按需同步生成：查询时若缺失则同步等待DeepSeek返回 |
| 数据完整性问题 | 创建缺陷后流转失败（403）但DRAFT已入库 | 用户反馈"403但刷新后出现在列表中" | 前端catch中回滚删除DRAFT；后端updateDefect校验前置 |
| React渲染问题 | LoginPage在render中调用navigate | 浏览器控制台报Transitioner警告 | 改为useEffect中navigate |
| 状态计数错误 | 多处文档写"8状态" | 代码审查 | 修正为9核心状态+REOPENED共10种 |

## 7. 人的关键决策

| 决策点     | 选项                     | 最终决策            | 决策理由               |
| ---------- | ------------------------ | ------------------- | ---------------------- |
| 数据库选型 | H2/MySQL/PG              | MySQL 8 + Docker    | 企业标准，评委认可度高 |
| 认证方案   | 无/JWT/Session           | JWT 3角色           | 展示完整工程能力       |
| AI集成深度 | 仅复盘/辅助建议/深度对话 | 辅助建议+面板审核   | 契合题目"人负责判断"   |
| 项目结构   | 分离/Monorepo/托管       | Spring Boot托管前端 | 部署极简               |
| Java版本   | 17/21                    | 21 LTS              | 最新LTS，本机已安装    |

## 8. 效率提升与问题

### 效率提升

- 后端58个Java文件一次性生成，编译通过率>95%
- E2E测试脚本自动执行全流程，比手工测试快10倍
- 文档模板化生成，减少重复劳动

### 主要问题

- AI首次生成的UI与用户期望差距大，需多轮迭代
- Playwright选择器在React动态渲染下容易超时
- 前后端联调时端口/代理配置需人工确认

## 9. 可沉淀为团队规范的经验

- **模块独立性原则**：每个模块可独立测试，通过接口通信
- **状态机模式**：用Map定义合法流转，集中管理校验规则
- **AI建议三态审核**：待审→采纳/拒绝/修改，完整记录人工决策
- **客户端筛选策略**：数据量<1000时优先客户端，体验更流畅
- **Sheet面板模式**：详情不用新页面，用滑出面板保持上下文
