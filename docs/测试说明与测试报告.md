# 测试说明与测试报告

## 1. 测试目标

验证缺陷分诊系统的核心业务规则和端到端流程是否按预期工作：
- 状态流转规则是否正确执行（9种状态+REOPENED共10种）
- 角色权限隔离是否生效
- AI建议生成和审核流程是否完整
- 知识沉淀是否自动触发

## 2. 测试范围

### 已覆盖
- 状态机流转规则（合法+非法流转）
- 必填字段校验（6项复现信息）
- 评估维度校验（5项影响评估）
- 优先级自动计算（加权公式）
- 角色权限控制（3角色边界）
- AI建议生成与审核
- 知识条目自动生成
- E2E全流程（创建→分诊→分析→修复→验证→关闭）

### 未覆盖
- 并发流转冲突（乐观锁）
- 大文件上传压力测试
- 1000+数据量性能测试
- DeepSeek API真实超时场景

## 3. 测试策略

| 测试类型 | 目标 | 工具/方法 |
|---|---|---|
| 单元测试 | 验证Service层业务逻辑 | JUnit 5 + Mockito |
| 集成测试 | 验证Controller+Service+Repository协作 | MockMvc + H2内存库 |
| E2E测试 | 验证用户操作全流程 | Playwright + Chromium |
| 手动验证 | UI交互、视觉一致性 | 浏览器实际操作 |
